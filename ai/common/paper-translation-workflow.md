# 论文翻译协作工作流

本文件定义了 AI 与人类用户协作进行长篇学术论文翻译与精排版的操作规范。在处理新的论文翻译任务时，一律按照本流程执行。

## 1. 准备阶段
- **获取源文件**：明确论文的纯文本（如 OCR 提取的 `txt`）或 PDF 源内容。
- **建立目标文档**：在对应的知识库目录下（如 `literature/translations/`）创建目标 Markdown 文件，命名推荐采用 Zotero Citation Key 结合后缀，如 `Huang2023-PIML-substructure-zh.md`。
- **确立大纲**：首先分析原文目录结构，在目标文档中预先写好各级标题（`# 1`，`## 1.1` 等），作为翻译框架和导航地图。

## 2. 逐节推进与双语对照 (核心循环)
为了保证翻译质量和排版准确性，严禁一次性翻译全文，以防丢失细节或因过长导致上下文截断。必须采用分块/分节推进的模式：
1. **AI 提交草稿**：针对当前小节，AI 给出**中英文对照**的翻译草稿，供用户审阅。
2. **预留图表占位符**：在译文草稿中，对于原文中的插图，AI 需留出明确的占位符（如 `<!-- 待插入图 1 -->`），并附带对应的中文图注（如 `<center>图 1：xxx</center>`）。
3. **用户审阅**：用户对当前小节的译文进行确认、提出修改意见，或提供该小节涉及的插图素材。
4. **写入文档**：在得到用户“确认无误”的反馈后，AI 负责将该小节的最终中文译文正式写入目标 Markdown 文件，并随后开启下一节的翻译。

## 3. 图表与公式处理规范
- **公式排版**：
  - **行内公式**：使用 `$ ... $` 语法，并注意公式与前后的中文字符之间保留适当间距。
  - **行间公式**：使用 `$$ ... $$` 语法。对于多行或带编号的公式，必须使用 `\begin{aligned} ... \end{aligned} \tag{X}` 格式以保证严格对齐和原文编号一致。
  - **矩阵与向量**：统一使用 `\boldsymbol{...}` 或 `\mathbf{...}` 进行加粗，保证数学符号规范。
- **图片处理**：
  - 用户上传图片后，AI 需将临时文件正确复制/移动至该文档相关的 `assets/` 目录下。
  - **图片命名规范**：`[CitationKey]_Fig[X].png`（例如 `Huang2023_Fig1.png`）。
  - **插入语法**：在文档中使用双链语法 `![[图片名.png]]` 或标准 Markdown `![图注](assets/图片名.png)` 替换之前的占位符，并在其下方紧跟 `<center>` 包裹的中文图注。

## 4. 全局终审 (Lint & Review)
当所有章节翻译完毕后，AI 需要主动对全篇 Markdown 文件执行一次全局扫描检查（Lint）：
- **完整性**：检查是否存在遗漏的英文段落或遗留的 `<!-- 待插入... -->`、`<!-- 待补完整中文译文 -->` 等占位标签。
- **格式**：检查所有 LaTeX 公式符号（`$`、`$$`）是否成对闭合，环境（如 `\begin{aligned}`）是否匹配。
- **连贯性**：检查图片编号顺序是否连续，小节标题编号是否按照原文逻辑递增。
- **最终汇报**：向用户汇报全篇审查结果，并交付最终文档。
