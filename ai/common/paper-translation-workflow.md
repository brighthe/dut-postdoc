# 论文翻译协作工作流

本文件定义了 AI 与人类用户协作进行长篇学术论文翻译与精排版的操作规范。在处理新的论文翻译任务时，一律按照本流程执行。

> 本流程在 Huang2024-PIML-datafree 的翻译中经过完整验证；文末「本机工具链速查」给出可直接复用的命令。

## 0. 新会话启动语（复制即用）

以后新开会话要翻译某篇论文时，只需说这一句，AI 即会读取本流程、定位 PDF、提取底稿并与你确认任务：

```
基于 paper-translation-workflow，和我一起翻译/完善 <CitationKey>-zh，是否了解任务？
```

- **`<CitationKey>`**：填 Zotero Citation Key 或对应阅读笔记名，如 `Huang2024-PIML-datafree`。
- **示例**：`基于 paper-translation-workflow，和我一起完善 Huang2024-PIML-datafree-zh，是否了解任务？`
- **两种场景通用**：
  - *从零翻译*：仓库尚无译文文件 → AI 会按 §1 建骨架后逐节推进。
  - *完善已有骨架*：已有 `-zh` 空框架 → AI 直接接着逐节补译。
- **可选补充**（不填也行，AI 会自行处理）：原文 PDF 路径（若不在默认 Zotero 库）、希望先从哪一节开始、特殊术语偏好。
- **收到后 AI 的默认动作**：读本流程 → 在 Zotero `storage/` 按文件名定位 PDF → `pdftotext` 提取底稿 → 核定章节目录 → 回复「任务理解 + 从摘要开始逐节给对照草稿」，等你确认再写入。

> 每节流程仍遵循 §2：AI 给中英对照草稿 + 核对点 → 你「确认」→ 才写入文件。你逐节回「确认」即可推进；图片按 §3.3 放到最后统一补。

## 1. 准备阶段

- **获取源文件**：优先从 Zotero 本地库取原文 PDF，再用 `pdftotext -layout` 提取纯文本到 scratchpad 临时目录供翻译参考。
  - Zotero 附件按**附件 key**（非条目 key）存于 `…/Zotero/storage/<附件key>/`；笔记里记的常是条目 key，可用文件名在 `storage/` 下检索定位真正的 PDF。
  - 提取的纯文本仅作**参考底稿**，不入库；矩阵/多行公式在纯文本里通常会错行，需后续重构（见 §3）。
- **建立目标文档**：在对应知识库目录（如 `literature/topology-opt/translations/`）创建目标 Markdown，命名用 Zotero Citation Key + `-zh` 后缀，如 `Huang2024-PIML-datafree-zh.md`。
- **确立大纲**：先从原文（或提取文本）扫描各级标题，在目标文档中**按原文目录**预写好各级标题（`# 1`、`## 1.1`…）作为翻译框架与导航图。骨架里每节先放 `> 待翻译。` 占位。
- **对齐姊妹篇**：若同一系列已有译文（如 Huang2023-zh），先看其排版、术语、图注、引用体例，保持全系列一致。

## 2. 逐节推进与双语对照（核心循环）

严禁一次性翻译全文，以防丢失细节或上下文截断。必须分块/分节推进：

1. **AI 提交草稿**：针对当前小节给出**中英对照**草稿供审阅。公式密集或过长的节可再拆成 2 个批次（如「§4 引言+4.1+4.2」「4.3+4.4」）。
2. **预留图表占位符**：插图留明确占位符 `<!-- 待插入图 X -->`，其下紧跟中文图注（图注排版见 §3）。
3. **列出核对点**：每批附一小段「核对点」——重构的公式、术语选择、疑似原文笔误等，方便用户聚焦审阅。
4. **用户审阅** → **写入文档**：得到「确认」后才把最终中文正式写入目标文件，随后开启下一节。

**翻译准则：**
- **引用体例忠实原文**：原文用 `(作者, 年份)` 就保留作者年份制，用 `[n]` 就保留数字制，不擅自改换。
- **术语一致**：与姊妹篇统一（如 EMsFEM=扩展多尺度有限元方法、multiscale shape function=多尺度形函数、substructure=子结构）；关键英文术语首次出现可中英并列。
- **原文笔误的处理**：发现原文错误（数字对调、拼写、漏词等）时，**不静默照抄也不无声改动**——按正确逻辑译出并加**译者脚注**说明原文情况（如表 3 数字对调、摘要漏「independent」）。

## 3. 图表与公式处理规范

### 3.1 公式排版
- **行内公式**：`$ ... $`，与前后中文之间留适当间距。
- **行间公式**：`$$ ... $$`；多行/带编号用 `\begin{aligned} ... \end{aligned} \tag{X}`，编号与原文一致。方程组用 `\begin{cases}`，矩阵用 `\begin{bmatrix}`/`\begin{pmatrix}`。
- **矩阵与向量**：统一 `\boldsymbol{...}`（或 `\mathbf{...}`）加粗。
- **公式重构与核对（关键）**：`pdftotext` 对矩阵/多行公式几乎必然错行，需凭领域知识重构，并在草稿核对点里**标注为「据 PDF 重构，待核对」**。定稿前**务必**把含重构公式的原文页用 `pdftoppm -png` 渲染成图片、用 Read 读图逐一核对（本次即以此纠正了式(10) `\mathcal{F}`、式(13) `J^j` 两处错误，并确认了式(22)大矩阵与附录转动矩阵）。

### 3.2 图注排版（重要，勿用 `<center>`）
- 图注/表标题若含行内公式 `$...$`，**不要用 `<center>…</center>`**：它是单行内联 HTML，Obsidian 不解析其内部的 `$...$`，公式会显示成源码。
- **正确做法**：用块级 `<div align="center">` 且**内部前后留空行**，Obsidian 才会把内部当 Markdown 解析、正常渲染公式，同时保持居中：

```
![[CitationKey_FigX.png]]

<div align="center">

图 X：……$\Omega_j$……

</div>
```

### 3.3 图片处理
- **顺序**：先完成全文文字与公式，**图片最后统一补**。
- **优先从 PDF 无损抽取**，不必让用户手动截图：
  - `pdfimages -list` 盘点内嵌图像与页码；排除首页期刊 logo 等非正文图。
  - 按「对象顺序 + 长宽比 / 页面位置」把图像映射到图号，并 Read 抽取结果中的几张交叉核验。
  - `pdfimages -png` 无损导出，按映射重命名。
- **命名规范**：`[CitationKey]_Fig[X].png`（如 `Huang2024_Fig3.png`）。
- **存放**：放到该文档所属方向的 `assets/` 目录（如 `literature/topology-opt/assets/`）；Obsidian 的 `![[名.png]]` 按文件名全库解析。
- **插入语法**：用 `![[图片名.png]]` 替换占位符，其下按 §3.2 的 `<div align="center">` 块写中文图注。

## 4. 全局终审（Lint & Review）

全部译完后主动执行一次全篇扫描，建议用脚本化检查：
- **完整性**：无遗漏英文段落、无残留 `待翻译`/`<!-- 待插入… -->`/`<center>`。
- **公式格式**：`$$` 成对（含 `$$` 的行逐行计数；注意反引号内的字面 `` `$$` `` 会造成计数为奇数，属误报）；`\begin{…}`/`\end{…}` 各环境数目相等；每行 `$` 个数为偶数；`\tag{N}` 编号连续无缺、无重复。
- **图表**：`![[…FigX.png]]` 数目 = 图注数；逐一确认引用的资产文件存在且非空；表标题、图号连续。
- **脚注**：每个 `[^name]` 均有定义且被引用。
- **收尾同步**：更新目标文档的元数据/状态与「译后检查清单」；**回填对应阅读笔记**（核心思路、学习对象、损失、实验、结论、与姊妹篇的关系表），并据实更新 `literature/_index.md` 的状态（`draft → done`）。
- **最终汇报**：向用户汇报审查结果并交付。

## 5. 本机工具链速查（Windows + MiKTeX + Zotero + Obsidian）

MiKTeX 自带 poppler 工具，路径 `C:\Program Files\MiKTeX\miktex\bin\x64\`：

- 定位 Zotero PDF：在 `…/Zotero/storage/` 下按文件名检索（附件 key ≠ 条目 key）。
- 提取全文：`pdftotext -layout "<pdf>" "<scratch>/raw.txt"`
- 盘点图片：`pdfimages -list "<pdf>"`
- 无损导出图片：`pdfimages -png "<pdf>" "<scratch>/img"`（输出 `img-000.png…`，前几张多为 logo）
- 渲染指定页核对公式：`pdftoppm -png -f <页> -l <页> -r 150 "<pdf>" "<scratch>/p"`，再用 Read 读图。

> 临时文件一律放 scratchpad，不入库；抽好的正式图片放对应 `assets/` 并按命名规范重命名。
