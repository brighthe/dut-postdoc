# 中国博士后科学基金第 80 批面上资助申请准备

## 3. 关键问题确认

### 3.1 是否必须入站之后才能申请？

**是。第 80 批面上资助应按“入站后、在站期间申请”理解。**

依据是官方指南对面上资助申请条件的表述：

- 面上资助面向博士后研究人员在站期间从事自主创新研究。
- 男性博士后研究人员进站 18 个月内可多次申请。
- 女性博士后研究人员进站 24 个月内可多次申请。
- 对在资助结果公布之前已办理出（退）站的博士后研究人员不予资助。

因此，面上资助不同于某些“拟进站”类联合资助或人才计划。第 80 批面上资助的申请前提应是已经完成博士后进站手续，并在系统中具有可申报身份。

需要向大连理工大学确认的细节：

| 问题 | 为什么要确认 |
|---|---|
| 进站手续完成日期 | 用于判断是否处于男性 18 个月 / 女性 24 个月申请窗口 |
| 系统中博士后身份是否已生效 | 决定是否能在管理信息系统中发起申请 |
| 是否属于“在职身份博士后” | 在职身份博士后不得申报，现役军人除外 |
| 第 80 批校内截止和审核流程 | 学校内部审核通常早于官方截止 |

### 3.2 第 80 批面上资助具体时间点

| 节点 | 时间 |
|---|---|
| 网上申报开始 | 2026-08-01 |
| 博士后申报截止 | 2026-08-31 |
| 设站单位审核截止 | 2026-09-07 |
| 评审截止 | 2026 年 11 月上旬 |
| 公示 | 2026 年 11 月下旬 |

实际操作上，不能按 2026-08-31 当天作为准备截止。建议以大连理工大学 / 学院内部截止为实际截止，并尽量在 2026-08-26 前后完成提交，预留系统填报、校内审核和驳回修改时间。

## 4. 申请资格硬条件

| 条件 | 官方口径 | 需要确认的个人事项 |
|---|---|---|
| 思想品德与科研诚信 | 具备良好思想品德、较高学术水平和较强科研能力，无科研失信行为 | 确认无科研诚信风险记录 |
| 进站时间 | 男性博士后进站 18 个月内可多次申请；女性博士后进站 24 个月内可多次申请 | 确认实际进站日期，并判断是否落在第 80 批可申报窗口内 |
| 累计在站时间 | 累计在站时间不超过 6 年 | 如有多站经历，需累计计算 |
| 获资助次数 | 每人每站只能获面上资助一次 | 确认本站是否已获面上资助 |
| 在职身份 | 在职身份博士后不得申报，现役军人除外 | 确认入站身份、人事关系和学校认定 |
| 项目属性 | 项目应具有基础性、原创性和前瞻性，具有重要科学意义和应用价值 | 申请书不能写成单纯工程开发或软件维护 |
| 涉密限制 | 项目非涉密，且为本人承担 | 题目、摘要、技术路线均按非涉密公开文本组织 |
| 派出项目限制 | 入选国家设立的博士后国（境）外派出项目人员不可申请 | 若涉及国境外派出或交流项目，需先确认兼容性 |

## 5. 申请材料与系统提交

第 80 批面上资助的核心材料是《中国博士后科学基金面上资助申请书》，由“中国博士后科学基金管理信息系统”在线生成并提交。

重点要求：

- “科研成果和奖励”填写代表申请人最高学术水平和科研成果的论文、专著、专利或奖励等，总数不超过 5 项。
- “项目信息”中，“研究基础”除外，不得出现申请人姓名、设站单位名称、合作导师姓名等可能泄露身份的信息。
- 网上申请开始前下载的申请书模板只能作为填报参考，不可作为正式申请书上传。
- 提交前应检查上传文档，不得开启文档加密保护。
- 如截止前需要修改已提交申请，需由审核单位逐级驳回。

## 6. 评审逻辑

面上资助由专家通讯评审确定资助人员。普通面上资助评审指标如下：

| 指标 | 分值 | 写作含义 |
|---|---:|---|
| 学术绩效 | 30 分 | 博士期间代表性论文、算法成果、软件原型和数值验证要服务于申请选题 |
| 创新能力 | 60 分 | 核心分值，必须把科学问题、方法创新、交叉特点写清楚 |
| 研究基础和条件保障 | 10 分 | 团队平台、导师方向、已有代码和实验条件用于支撑可行性 |

对当前申请来说，最大写作原则是：**不要把博士后三年总体计划原样塞进面上申请，而要压缩成一个可评审、可完成、可验证的科学问题。**

## 7. 当前最合适的选题压缩方向

| 候选方向 | 推荐程度 | 申请书主问题 | 风险控制 |
|---|---|---|---|
| PIML + 多尺度结构分析 | 高 | 如何学习并可靠嵌入子结构等效算子，以降低多尺度结构分析成本 | 避免写成“用神经网络替代有限元”的泛泛叙述 |
| Matrix-Free 高性能有限元 | 高 | 如何通过无矩阵算子作用、Krylov 迭代和预条件子提升大规模结构分析效率 | 避免只写工程加速，要突出数值方法问题 |
| PIML + Matrix-Free 融合 | 中-高 | 如何把局部等效算子学习与全局无矩阵求解连接起来，形成可扩展分析框架 | 范围容易过大，需要明确只做一个关键环节 |
| MMC/MMV 显式几何分析接口 | 中 | 如何从显式几何生成可靠积分点、局部算子和快速分析接口 | 当前原型主要证明前向接口，不宜承诺完整优化闭环 |

### 暂定优先选题

建议优先从前两项中选择一个：

1. **面向复杂多尺度结构的 PIML 等效算子学习与快速分析方法**
2. **面向大规模结构分析的 Matrix-Free 有限元算子与 Krylov 快速求解方法**

如果希望更贴近博士后整体计划，可采用第三类融合题目，但申请书中必须收窄到一个关键接口或一个关键算子问题。

## 8. 可用研究基础

| 基础材料 | 可支撑的申请叙述 | 需注意边界 |
|---|---|---|
| 入站答辩 Part 2 博士后计划 | 说明研究方向、问题来源和三年路线 | 不能直接照搬，需要压缩为面上项目 |
| PIML 子结构多尺度原型 | 支撑等效算子学习、子结构刚度预测和多尺度分析可行性 | 需明确误差指标对应的是局部等效刚度，不是最终结构优化误差 |
| Matrix-Free 原型与 benchmark | 支撑无矩阵算子作用、MatVec/CG 一致性和高性能实现经验 | 需避免宣称端到端大系统已完成 |
| MMC/MMV cut-cell 前向接口 | 支撑显式几何到积分数据入口的已有基础 | 当前只证明几何到积分点接口，不证明求解闭环 |
| 郭旭院士团队与平台背景 | 支撑拓扑优化、MMC/MMV、PIML、工业软件场景 | 平台支撑应服务于科学问题，不应成为申请书主体 |
| 计算数学 / 高性能数值实现积累 | 支撑先进离散、快速算法、预条件子和高性能求解 | 需要转化成申请书中的“研究基础”和“条件保障” |

## 9. 申请书准备框架

### 9.1 题目

要求：窄、硬、可评审，避免同时堆满 PIML、Matrix-Free、MMC/MMV、GPU、拓扑优化等所有关键词。

可先准备 3 个版本：

| 版本 | 题目草案 | 适用情形 |
|---|---|---|
| A | 面向复杂多尺度结构的 PIML 等效算子学习与快速分析方法 | 偏 PIML 和多尺度 |
| B | 面向大规模结构分析的 Matrix-Free 有限元算子与快速求解方法 | 偏数值线性代数和高性能有限元 |
| C | 面向拓扑优化迭代分析的多尺度等效算子复用与无矩阵求解方法 | 偏博士后整体路线，但需控制范围 |

### 9.2 摘要

摘要建议包含四句话：

1. 问题背景：多尺度结构 / 拓扑优化迭代分析存在重复求解、尺度耦合或矩阵存储成本高的问题。
2. 核心瓶颈：传统方法在局部等效、全局求解或迭代复用上存在效率和可扩展性限制。
3. 拟解决方法：提出 PIML 等效算子学习、Matrix-Free 算子作用或二者接口方法。
4. 预期结果：形成可验证算法、原型程序和典型算例，服务高效结构分析。

### 9.3 研究内容

建议控制为 3 个研究内容：

| 研究内容 | 写作目标 |
|---|---|
| 内容 1：问题建模与误差 / 算子机制分析 | 说明不是简单套模型，而是在分析关键数值机制 |
| 内容 2：核心算法构造 | 说明 PIML 等效算子、Matrix-Free 算子或接口方法如何构造 |
| 内容 3：数值验证与原型实现 | 说明用哪些算例、指标和原型验证方法有效性 |

### 9.4 创新点

创新点建议写 3 条，不宜过多：

- 从“局部等效算子学习”或“无矩阵算子作用”切入，提出面向结构分析的可复用数值表示。
- 建立学习型局部算子与全局有限元 / 迭代求解过程之间的接口机制。
- 形成面向多尺度或拓扑优化迭代场景的快速分析原型，并用典型算例验证精度和效率。

### 9.5 预期成果

可写为：

- 形成 1 套面向多尺度结构或大规模结构分析的快速算法框架。
- 完成若干典型算例验证，给出精度、效率和可扩展性指标。
- 形成可复现实验代码或原型程序。
- 发表 / 投稿相关论文，支撑后续青基或特别资助申请。

## 10. 代表性成果准备

面上资助“科研成果和奖励”最多 5 项。建议按“直接相关优先”原则排序。

| 类别 | 候选内容 | 当前动作 |
|---|---|---|
| 博士论文核心成果 | 高阶多分辨率、有限元、拓扑优化或相关数值方法成果 | 整理 1 段与申请题目直接相关的贡献说明 |
| 代表性论文 | 已发表 / 已接收 / 在投论文 | 建立题名、作者、期刊、状态、本人贡献表 |
| 软件 / 原型 | PIML、Matrix-Free、MMC/MMV 相关原型 | 确认系统中是否可填，若不可填则转化为研究基础叙述 |
| 专利 / 奖励 | 若有与计算力学、软件、算法相关成果 | 只放与申请题目强相关的项目 |
| 项目参与 | 博士期间或团队项目经历 | 可用于研究基础，不一定放入代表性成果 |

## 11. 倒排计划

以 2026-08-31 官方博士后申报截止为界，建议不按最后一天提交。

| 时间 | 目标 | 产出 |
|---|---|---|
| 2026-07-07 至 2026-07-14 | 确认资格和校内流程 | 入站身份、校内截止、学科归属、申请系统入口 |
| 2026-07-15 至 2026-07-21 | 确定主选题 | 题目 1 个、备用题目 1 个、300 字摘要 |
| 2026-07-22 至 2026-07-31 | 完成申请书初稿 | 研究内容、创新点、技术路线、研究基础初稿 |
| 2026-08-01 至 2026-08-10 | 系统填报与第一轮修改 | 在线生成申请书，完成格式检查 |
| 2026-08-11 至 2026-08-20 | 导师 / 同行反馈 | 根据反馈收紧科学问题、创新点和可行性 |
| 2026-08-21 至 2026-08-25 | 定稿前检查 | 匿名信息检查、代表性成果核对、附件和系统字段核对 |
| 2026-08-26 前后 | 尽量完成提交 | 预留审核驳回、修改和系统拥堵时间 |
| 2026-08-31 | 官方博士后申报截止 | 不建议拖到当天首次提交 |

## 13. 风险清单

| 风险 | 表现 | 规避方式 |
|---|---|---|
| 题目太大 | 把 PIML、Matrix-Free、MMC/MMV、GPU、拓扑优化全部放入一个面上申请 | 聚焦一个主对象和一个核心瓶颈 |
| 工程化过重 | 写成软件开发或算力加速，不像基础研究 | 强调数值机制、误差、算子、可扩展性等科学问题 |
| 匿名评审泄露身份 | 项目信息中出现申请人、设站单位、合作导师等信息 | 提交前专门做匿名检查 |
| 承诺超过已有基础 | 宣称完整闭环系统已经完成 | 用“已有原型支撑、拟进一步研究”表述 |
| 校内截止遗漏 | 只盯官方 2026-08-31 | 以学校 / 学院内部截止为实际截止 |
