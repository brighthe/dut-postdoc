## 帧 3 · 方法二 · 高阶多分辨率框架

**核心一句话**：通过分析网格、密度积分网格与设计变量网格三层解耦，在控制分析自由度规模的同时释放高分辨率设计表达能力。

**配图**：`multires_framework` 三层网格示意（主图）+ 可选 `mbb_mtop_k4`（高分辨率构型）作小图对照。

**三层网格（精简三栏）**
- $\mathcal{T}_h$ 高阶位移网格：较粗高阶单元，控制分析自由度规模；
- $\mathcal{T}_\rho$ 密度积分网格：子单元密度积分，保证刚度组装一致性；
- $\mathcal{T}_d$ 设计变量网格：承载高分辨率设计变量，提升拓扑表达。

**表达优势数据（可放图注一行）**：MBB 梁，STOP($k=1$) $\rho_{\text{dof}}=600$ vs MTOP($k=4$) $\rho_{\text{dof}}=9600,\ u_{\text{dof}}=19762$ —— 同分析规模下设计自由度大幅提升、构型更清晰。

**底部结论条**：三层解耦释放高分辨率构型表达；应力评估仍依赖位移后处理 $\Rightarrow$ 引出方法三。

**讲稿要点**：核心卖点是“解耦”。原“密度映射鲁棒性”“L 型支架应力约束应用”两帧删去，应力话题交给方法三。

## 帧 4a · 方法三 · 任意次胡张混合有限元

**核心一句话**：以应力为独立未知量、构造 $H(\mathrm{div})$ 协调对称应力空间的任意次胡张混合元，嵌入变密度拓扑优化闭环，直接获得高精度应力场。

**配图**：抗闭锁对比 `bearing_disp_nu4999` vs `bearing_mixed_nu4999`（二维轴承，近不可压 $\nu_0=0.4999$）；如版面够，附应力场 `stress_vm_disp` vs `stress_vm_mixed` 小图。

**左：方法（精简）**
- 以应力 $\boldsymbol\sigma$ 为独立未知量，构造 $H(\mathrm{div})$ 协调对称张量有限元空间（北大胡俊教授方法，完成程序实现并引入拓扑优化）；
- 基于单纯形晶格统一实现任意阶次；每步迭代同时求解 $\boldsymbol\sigma_h,\boldsymbol u_h$，直接由 $\boldsymbol\sigma_h$ 构造柔顺度目标与局部应力约束。

**右：两项代表性能力（数据）**
- **抗体积闭锁**：近不可压 $\nu_0=0.4999$ 下，标准位移法刚度高估、构型异常（$c\approx55.78$）；胡张混合法构型稳定（$c\approx102.79$）。
- **应力场可靠评估**：位移法应力场单元间非物理跳跃；混合法基于协调应力空间，应力场平滑连续、约束评估更可靠（悬臂梁 V 型承载构型）。

**底部结论条**：实现"胡张混合分析 → 变密度拓扑优化闭环"系统拓展，为高精度应力驱动拓扑优化奠定基础。

**讲稿要点**：本页重点讲从位移法到混合法的必要性，以及近不可压算例中的抗闭锁能力；底部结论顺势引出应力约束优化。

## 帧 4b · 方法三拓展 · 应力约束拓扑优化结果

**核心一句话**：在局部应力约束体积最小化问题中，胡张混合法直接求解协调应力场，相比标准位移法后处理应力，更适合作为应力约束拓扑优化的可靠分析基底。

**配图**：博士论文第五章图 5.12 四宫格：`stress_topo_disp`、`stress_vm_disp`、`stress_topo_mixed`、`stress_vm_mixed`，按“标准位移法一行 / 胡张混合法一行；拓扑构型 / 归一化 von Mises 应力两列”排列。

**底部结论条**：胡张混合元把高精度应力场直接纳入优化闭环，为局部应力约束拓扑优化提供可靠分析基底。

**讲稿要点**：这页不再讲方法细节，只用第五章结果说明“直接求解协调应力”在应力约束问题中的价值：应力场更平滑，约束评估更可靠。

## 帧 5a · SOPTX 软件平台

**核心一句话**：以"分析—优化解耦"为核心构建模块化 SOPTX 平台，支持 AD、多后端与 CPU/GPU 异构。

**版式建议**：上"架构 + 关键技术"双栏，下 GPU 加速表 + 结论条。

- **架构三层**：用户交互层 / 核心算法层（插值-分析-正则化-优化闭环）/ FEALPy 基础后端层；
- **关键技术**：高效张量化矩阵组装、PyTorch/JAX 自动微分灵敏度、NumPy/PyTorch/JAX 多后端与 CPU/GPU 异构；
- **性能亮点（GPU 加速表，三维悬臂梁）**：总耗时 2789.9 s → **636.7 s**，端到端 **≈4.38×**；稳态平均迭代 21.09 s → 4.84 s，**≈4.36×**；
- 多后端构型/收敛一致（NumPy/PyTorch/JAX）；AD 与手动微分结果一致、效率相当、扩展更便捷（口头一句）。

**底部结论条**：模块化解耦 + AD + 多后端异构，为高阶有限元拓扑优化提供高效可扩展软件支撑。

**讲稿要点**：讲"解耦 + GPU 4.38× + 多后端一致"三点即可，~40 秒。

## 帧 6 · 科研计划总览与承接关系

**核心一句话**：博士后科研计划围绕既定两大方面展开：一是 PIML 增强多尺度结构分析与 Matrix-Free 高性能求解，面向大规模结构分析的效率与存储瓶颈；二是 MMC/MMV 显式拓扑优化先进数值分析，面向显式几何与高阶/混合离散方法的融合。

**承接关系（建议用箭头表）**

## 帧 7 · 方向一 · PIML 增强的子结构多尺度前向分析原型

**核心一句话**：PIML 学习"局部材料分布 → 子结构多尺度形函数 / 等效刚度"的映射，并嵌入"粗网格密度 → 子结构等效刚度 → 全局粗尺度求解"的前向分析流程。

**配图**：`piml_baseline.pdf`（待生成）——上方三步横向展示"宏观粗网格密度 → 选中粗单元 → 子结构细网格密度"，再在中间偏右位置阶梯式下接"PIML 预测 → 接口缩聚 / 等效刚度 → 全局粗尺度求解"；PIML 与最终求解框用浅蓝底突出方法核心与流程出口。

**主线 1：PIML 多尺度形函数预测与粗尺度等效刚度**
- 以粗网格单元 / 子结构内部细尺度材料分布为输入，离线学习多尺度形函数或等效局部算子；
- 在线嵌入有限元分析与拓扑优化流程，减少重复局部求解，并关注对称性、正定性与能量一致性。

**建议放入的数值结果表（2026-07-02 已回填 SOPTX 实测，分支 `codex/piml-multiscale-prototype`，benchmark_piml_forward）**
| 验证环节 | 数值指标 | 当前结果 |
|---|---|---|
| 前向规模 | 粗网格 $8\times8$（64 子结构）；局部细网格 $5\times5$ / $10\times10$ | 全尺度 DOF `3362 / 13122` → 接口 DOF `1314 / 2754`（降维 `2.6x / 4.8x`） |
| 缩聚正确性 | $\|K^{\mathrm{cond}}-S^{\mathrm{Schur}}\|_F/\|S^{\mathrm{Schur}}\|_F$（vs 全尺度全局 Schur 补） | $5\times5$: `1.4e-15`；$10\times10$: `2.6e-15`（机器精度，目标 $<10^{-10}$ 达成） |
| 接口求解 | 残差 $\|K U_b-F_b\|/\|F_b\|$；接口解 vs 全尺度直解 | 残差 `2.1e-13 / 3.4e-13`；对直解误差 `9.4e-14 / 4.3e-12`（细尺度恢复同量级） |
| PIML $K_s$ 预测 | $\|\widehat K_s-K_s\|/\|K_s\|$；对应 `piml_pred_error.pdf` | 目标 $10^{-3}$--$10^{-2}$（T4b 极小网络待训练；Mock 解析映射对照 `2.8e-2 / 3.1e-2`） |

**讲稿要点**：这页只回答"PIML 预测什么、前向验证怎么支撑"。用流程图支撑"局部材料分布 → 多尺度形函数 / 等效刚度 → 粗尺度求解"理解链；数值口径：**静力缩聚数学上精确**——逐子结构 $K_s^j$ 组装的接口方程与全尺度全局 Schur 补机器精度一致（~1e-15），接口解与全尺度直解一致（~1e-12），全局求解规模降至接口自由度；PIML 改变的是"如何快速获得 $K_s^j$"（预测替代内部自由度消元），预测误差行待 T4b 极小网络训练后替换（算例：矩形悬臂，非均匀密度场，E=1、ν=0.3 平面应力；数据见 soptx `ai/common/progress-frame7_piml.md`）。

## 帧 8 · 方向一 B · Matrix-Free 高性能求解原型

**核心一句话**：Matrix-Free 不改变有限元离散模型，只将"组装并存储全局刚度矩阵"替换为"按需计算算子作用 $y=Kx$"，为大规模 Krylov 求解和 GPU/MPI 并行提供入口。

**配图**：Matrix-Free 算子作用流程图 + 正确性/性能/扩展路线表。流程图建议为：

```text
全局向量 x
  -> gather 单元自由度 x_e
  -> 局部张量收缩 action
  -> scatter-add 得到 y=Kx
  -> Krylov / CG 求解 KU=F
```

**主线 2：全局算子 Matrix-Free 高性能求解**
- 以**不显式组装、不持久存储全局矩阵**为核心，降低大规模结构分析中的内存需求；
- 结合 Krylov 迭代、Jacobi/Chebyshev 预条件、GPU 后端与 MPI 并行基础，提升超大规模结构分析的计算效率和并行可扩展性。

**建议放入的数值结果框架（2026-07-02 已回填真实 benchmark 口径）**
| 验证环节 | 数值指标 | 当前结果 |
|---|---|---|
| MatVec 等价性 | `||MF(x)-Kx|| / ||Kx||` | 2D/3D 误差约 `1e-15`–`1e-13`；standard contraction 不形成局部 `K_e` |
| 状态方程闭环 | CG 收敛、残差、`u_MF` 与组装式结果一致性 | 小规模 NumPy / PyTorch CPU / PyTorch CUDA 三档一致，残差约 `1e-11`–`1e-10` |
| 单卡 GPU MatVec | PyTorch CUDA vs NumPy CPU，大规模 2D benchmark | `ndof≈1.32e5` 时 MatVec 约 `11.9×`；内存估计 `42.1 MB → 4.0 MB` |
| GPU/MPI + 预条件基础 | 前期 PA / Matrix-Free 算子级端到端求解与预条件验证 | `650` 万 DOF hex，GPU + OpenMPI，`mpirun=1`–`32`，相对基线总时间约 `3.72×`–`12.74×`；P2 tet：Jacobi / Chebyshev 下 64 核 CPU 约 `1.20×`–`1.21×`，GPU 多数配置 `4×+` |

**讲稿要点**：这页回答"全局矩阵怎么不组装、正确性怎么保证、当前性能证据到哪里，以及并行/预条件工程基础是否具备"。已有结果应说成：入站前的前期基础验证已经打通无矩阵状态方程链路，2D/3D MatVec 与组装式路径机器精度一致，小规模三后端状态方程一致，单卡 GPU MatVec 随规模上升出现加速趋势；同时，前期 PA / Matrix-Free 算子验证说明我已经具备 GPU+MPI 并行求解和 Jacobi / Chebyshev 预条件求解的工程基础。注意边界：这些结果证明的是入站前的基础工作与并行/预条件算子能力，不等同于博士后方向一的 PIML × Matrix-Free 一体化系统已经完成。

## 帧 9 · 方向一 C · PIML × Matrix-Free × GPU 融合路线

**核心一句话**：PIML 负责按需生成逐子结构等效算子，Matrix-Free 负责全局不组装求解，GPU 负责进一步提升并行效率，三者共同形成面向大规模拓扑优化的软件分析后端。

**配图**：融合路线图，突出"局部预测"与"全局无矩阵求解"的咬合：

```text
宏观密度 rho
  -> 逐子结构 PIML 预测 K_s^j
  -> 不组装全局 K
  -> y = sum_j A_j^T K_s^j A_j x
  -> Krylov + 预条件 + GPU 并行
  -> 可复用分析内核与高性能求解模块
```

**融合机制（建议放 4 条）**
- **PIML 输出**：逐子结构等效刚度 $\widehat K_s^j$，替代局部细尺度重复求解；
- **Matrix-Free 作用**：在 Krylov 迭代中按需计算 $y=\sum_j A_j^T\widehat K_s^jA_jx$，不形成全局缩聚矩阵；
- **非线性扩展**：依托团队 PIML 在非线性问题上的进展，后续纳入软件分析后端；
- **软件目标**：沉淀可复用分析内核与高性能求解模块，后续对接 GPU 并行与工业软件架构。

**长期牵引（一句，建议小字/底部结论）**：面向桌面级 $10^9$ 细网格等超大规模计算，按"局部预测 + 全局无矩阵 + GPU 并行"倒推算法与软件架构。

**讲稿要点**：上一页 PIML 给出局部等效算子，上一页 Matrix-Free 给出无矩阵算子作用；这一页把两者接起来。郭老师提出的桌面级 $10^9$ 细网格目标作为长期牵引口头说明，不写成短期承诺。
