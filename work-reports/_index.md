# 博士后工作汇报归档

> 本目录保存面向导师或合作团队的工作汇报技术材料。每次汇报独立成页，并在同一页面中经历“准备中 → 已汇报 → 跟进完成”的生命周期。

## 事实源分工

| 内容 | 权威位置 | 本目录的处理方式 |
|---|---|---|
| 科研技术事实、原型证据、研究路线、汇报要点、会后技术决定与行动项 | `dut-postdoc` | 在对应汇报页中沉淀并链接技术来源 |
| 真实微信消息、逐字或近逐字交流、约见安排、礼节和关系状态 | `heliangos/wechat` | 不复制正文；需要时只标明外部事实源 |
| 研究院项目任务、分工和实时状态 | `dut-institute-work` 对应计划页 | 只引用当前状态，不另建并行任务账 |

## 生命周期

| `status` | 含义 | 更新条件 |
|---|---|---|
| `preparing` | 会前准备中，尚未实际汇报 | 新建汇报页时使用 |
| `reported` | 已经实际汇报，仍有待跟进行动 | 会后补充研究结论与行动项时更新 |
| `follow-up-done` | 本次汇报产生的行动项已完成 | 所有行动项完成后更新 |

未实际发生的汇报不得标记为 `reported`；会前计划、候选路线和拟请教问题不得写成老师已经确认的结论。

## 新建流程

1. 复制 [[../assets/templates/advisor-work-report]]，放入对应汇报对象目录。
2. 文件名采用 `YYYY-MM-主题.md`；同月多次汇报用不同主题区分，实际日期写入 `meeting_date`。
3. 会前填写技术事实、状态边界、汇报要点和待请教问题，状态保持 `preparing`。
4. 会后只追加研究结论、老师的技术指导和行动项；真实沟通过程回到 `heliangos/wechat` 记录。
5. 同步对象目录 `_index.md`、本页、[[../index]] 和 [[../log]]；若会后结论影响研究路线，再检查并更新关联的 `research/` 页面。
