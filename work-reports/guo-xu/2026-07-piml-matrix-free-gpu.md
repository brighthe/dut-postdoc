## 一、汇报要点（结论先行）

建议开场先用三句话交代清楚：

1. 入站前我已经分别完成了 PIML 子结构局部算子原型和 Matrix-Free 状态方程原型，并且此前已有 GPU/MPI、Krylov 和预条件子方面的工程经验。
2. 当前这三块还没有正式接成 PIML × Matrix-Free × GPU 一体化系统，缺口主要在子结构算子接口、结构保持、预条件和完整优化闭环。

## 二、已经完成的技术工作

### 2.1 PIML：局部子结构前向链路

已经完成：

- 跑通“局部细尺度密度 → 子结构静力缩聚 → 等效刚度 $K_s^j$ → 全局接口缩聚方程 → 接口求解/细尺度恢复”的基本链路。
- 精确缩聚与全尺度 Schur 补误差为 $10^{-15}$ 量级；接口解与全尺度直解保持 $10^{-14}$–$10^{-12}$ 量级一致。
- 训练极小 MLP 直接预测 $K_s$，部署算例上逐子结构相对预测误差均值为：
  - $5\times5$：$1.6\times10^{-3}$；
  - $10\times10$：$8.2\times10^{-3}$。
- 两档均优于只用局部平均密度的 Mock 对照。

需要主动说明：

- `1.6e-3 / 8.2e-3` 是局部 $K_s$ 预测误差，不是位移、柔顺度、灵敏度或最终拓扑误差。
- 尚未完成结构保持参数化、多尺度灵敏度和完整拓扑优化闭环。

### 2.2 Matrix-Free：全局无矩阵状态方程基础

已经完成：

- 在积分点直接做 $B^TDBx$ contraction，不形成全局 $K$，也不预先形成单元 $K_e$。
- Matrix-Free MatVec 与显式矩阵乘的相对误差达到 $10^{-15}$–$10^{-13}$。
- Matrix-Free CG 已跑通小规模 $KU=F$ 求解闭环，残差约 $10^{-11}$–$10^{-10}$。
- NumPy、PyTorch CPU、CUDA 三后端结果一致。
- 13.2 万 DOF 下单次 GPU MatVec 约 $11.9\times$，内存估计从 42.1 MB 降至 4.0 MB。

需要主动说明：

- $11.9\times$ 是单次 MatVec 加速，不是完整 solve 加速。
- 当前局部算子仍是常规有限元单元作用，尚未正式换成 PIML 子结构 $K_s^j$。

### 2.3 GPU/MPI/预条件子：已有独立工程经验

本人编写的 `mfleo` PA / Matrix-Free 包已有：

- 650 万 DOF 的 GPU/MPI 端到端 CG；1–32 个 MPI 进程配置下，相对同规模 `MFEM PA` 基线约 $3.72\times$–$12.74\times$。
- P2 tet 的 Jacobi、Chebyshev 预条件子测试；CPU 约 $1.20\times$–$1.21\times$，GPU 多数配置约 $4\times+$。

准确定位：这些结果证明我有 GPU/MPI、Krylov 和基础预条件子工程经验，可作为后续迁移基础；它们不是当前 soptx/PIML 一体化结果。

## 三、当前事实边界与真正缺口

| 层次 | 当前已有 | 当前缺口 |
|---|---|---|
| PIML 局部层 | 局部密度 → $K_s$，有真实预测误差 | SPD/能量一致、误差传播、灵敏度和优化闭环 |
| Matrix-Free 全局层 | 真无矩阵 MatVec、CG、三后端一致 | 子结构 $K_s$ 接口、大规模收敛和预条件 |
| GPU/HPC 层 | soptx 算子级趋势；`mfleo` 独立工程结果 | PIML 批量推理与子结构 Matrix-Free 融合 |
| 软件层 | 有原型和独立包 | 可复用的一体化分析后端 |

核心问题可以压成一句：

> 现在不是三块都从零开始，而是各自已有基础；下一步的科学和工程增量，是让 PIML 预测的局部算子在保持数值结构的前提下，可靠进入全局 Matrix-Free Krylov 求解，并在 GPU 上形成端到端收益。

## 四、后续设想：近期推进路线

### 4.1 建议顺序

```text
研究院求解器与性能基线
  -> 精确 K_s 的子结构级全局 Matrix-Free
  -> Krylov + 基础预条件子
  -> 替换为 PIML 预测 K_s，检查结构与误差传播
  -> PIML 推理 + 局部算子 GPU 批处理
  -> 灵敏度、优化闭环、多节点和软件集成
```

### 4.2 为什么按这个顺序

- 先用精确 $K_s$，可以隔离 Matrix-Free 接口错误与 PIML 预测误差。
- 先建立 Krylov/预条件基线，才能判断预测算子是否破坏收敛。
- 先稳定数据接口和正确性，再做 GPU，性能数字才可解释。
- 灵敏度和优化闭环放在接口可靠之后，避免同时引入过多误差源。

### 4.3 第一批可交付结果

1. 研究院 Hypre/PETSc KSP 代表算例的收敛性与性能基线。
2. 精确 $K_s$ 下，子结构级 Matrix-Free MatVec 与显式缩聚矩阵乘的机器精度对照。
3. none/Jacobi/block Jacobi 等基础配置的残差、迭代数、时间和内存表。
4. 换入 PIML $\widehat K_s$ 后的对称性、最小特征值、谱性质、全局位移误差和 Krylov 收敛报告。
5. 批量 PIML 推理、局部算子作用和 scatter-add 的 GPU profiling。

### 6.2 我可以提供的具体切入角度

不从“哪个网络最先进”回答，而是建立六层筛选：

1. **学习对象**：全局结果、局部场、形函数还是 $K_s$？
2. **数据拓扑**：固定向量、规则栅格、坐标连续还是非结构网格？
3. **数据成本**：监督标签、data-free 还是混合训练？
4. **物理结构**：对称、正定、刚体模态和能量一致性如何保证？
5. **求解器影响**：模型误差是否破坏 CG/GMRES、灵敏度和优化收敛？
6. **部署目标**：训练成本、GPU 吞吐、显存和端到端 solve 时间如何权衡？

建议的最小研究切口是：

> 在同一个 PIML 局部等效算子任务上，比较 MLP、DeepONet/operator learning 及其他候选表示，不只比较预测误差，还比较 SPD 保持、Krylov 收敛、全局响应误差和 GPU 部署成本，形成模型选型 Pareto 图与决策规则。

其中：

- MLP 已有本地原型证据。
- DeepONet/operator learning 已有团队 Huang 2024 文献依据，但本地尚未复现。
- CNN、面向非结构网格的 GNN 等只能作为待验证候选，不能说已确定采用。

### 6.3 当前合作边界

当前只能说“刘老师提出了值得进一步讨论的技术痛点，我已形成初步问题框架，正在准备确认具体场景”。在刘老师进一步明确具体物理问题、数据条件、软件环境和评价目标前，不能表述为已确定合作。
